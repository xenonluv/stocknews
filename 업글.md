# 업그레이드 예정 — 분봉(reignition) NXT 통합(UN) 전환

> 작성 2026-06-21(일). NXT 통합 전환의 **마지막 미완 1건**. 거래일(개장) 실측이 필요해 보류된 부분.
> 선행 작업(가격=J / 거래대금·수급=UN, 폭발 1천억→1,500억 등)은 완료·검증됨 — `패치0618.md` 2026-06-21 항목 참조.

## 배경 — 왜 분봉만 남았나

거래대금은 KRX+NXT 통합(UN)으로 전환했지만 **분봉(`minute_bars_today`)만 아직 J(KRX 단독)** 이다.
이유: NXT는 **프리마켓(08:00~08:50)·애프터마켓(15:40~20:00)** 등 정규장 밖 시간대가 있어, UN 분봉에
장 밖 봉이 섞이면 "당일 재반등 10분봉" 로직이 오탐날 수 있다. 일요일엔 라이브 분봉이 없어 검증 불가 →
KIS 문서의 "시간대 동일" 추정만 믿지 않고 **거래일에 직접 확인 후 전환**하기로 보류.

### 현재 상태(준비만 됨)
- `kis_client.minute_bars_today(code, until, market="J")` — 기본 **J**. `market` 파라미터는 추가됨(UN 수용 가능).
- **정규장 시간창 가드 적용**: `SESSION_OPEN("090000") <= t <= SESSION_CLOSE("153000")` 봉만 채택
  (`kis_client.py` minute_bars_today 내부). → UN으로 돌려도 장 밖 봉은 이미 걸러지도록 안전판 마련.
- `radar.py:586` 은 `kis.minute_bars_today(code)` 로 호출 → **현재 J로 동작**.
- 임계 `REIGNITION_VALUE_10M=30억`(`radar.py:60`), `re_value` 스케일 30~300억(`radar.py` breakdown) **미변경**.

## 월요일(다음 개장일) 패치 방향

### STEP 1 — 장중 실측 검증 (장 열린 시간, ~10:00 이후 권장)
거래일에 아래를 실행해 **UN 분봉이 정규장만 깔끔히 들어오는지** 확인한다.

```bash
cd /Users/jinjin/stocknews
# (A) UN 분봉 시각 분포 — 장 밖(09:00 이전 / 15:30 이후) 봉이 섞이는지, 가드가 거르는지
python3 - <<'PY'
import sys; sys.path.insert(0,'scripts'); import kis_client as kis
for mkt in ("J","UN"):
    b = kis.minute_bars_today("010690", market=mkt)   # 가드 적용된 결과
    ts = [x["time"] for x in b]
    print(f"[{mkt}] 봉수={len(b)} 최초={ts[0] if ts else '-'} 최후={ts[-1] if ts else '-'}")
    # 가드 우회 raw 확인이 필요하면 SESSION_OPEN/CLOSE를 잠시 넓혀 재측정
PY
```
**판정 기준**:
- UN 봉의 최초/최후 시각이 09:00~15:30 안에 있어야 함(가드 정상).
- UN 봉 수 ≈ J 봉 수(정규장 분 수 동일), UN 10분봉 거래대금 ≥ J(거래량 증가).
- 만약 UN 응답에 장 밖 봉이 대량 섞이거나 시각 체계가 다르면 → **전환 보류, 가드 로직 재점검**.

### STEP 2 — 10분봉 거래대금 UN/J 배율 실측 (임계 재산정 근거)
일봉 배율(~1.5)과 **장중 10분봉 배율은 다를 수 있으므로** 별도 측정한다.
`nxt_calibration.py`를 확장하거나 장중에 아래로 표본 수집:
```bash
# 재반등 후보(폭발 이력) 몇 종목의 당일 10분봉을 J·UN으로 합성해 value 배율 비교
# (radar.aggregate_minute_bars 재사용: val = close*vol → 10분 합산)
```
- 결과 배율 r10을 보고 `REIGNITION_VALUE_10M`·`re_value` 스케일을 r10으로 정함(아래 STEP 3 기본값은 ×1.5 잠정).

### STEP 3 — 코드 전환 (STEP 1·2 통과 시에만)
1. `scripts/radar.py:586`
   - `bars = kis.minute_bars_today(code)` → `bars = kis.minute_bars_today(code, market="UN")`
   - (또는 `minute_bars_today` 기본값을 "UN"으로. 단 셀프테스트·타 호출 영향 점검)
2. `scripts/radar.py:60`
   - `REIGNITION_VALUE_10M = 3_000_000_000`(30억) → **`4_500_000_000`(45억)** (또는 STEP2의 r10 반영)
   - argparse `--reignition-value-10m` help 문구도 갱신(`radar.py:920` 부근)
3. `scripts/radar.py` breakdown `re_value`
   - `(re_value_max - 30) / 270 * 12` → **`(re_value_max - 45) / 405 * 12`** (30~300억 → 45~450억, ×1.5)
4. `re_body`·`re_count`는 거래대금 무관(몸통%·봉 개수) → **불변**.
5. ⚠ **`minute_bars_today`의 `got==0` 조기 break 보강 (UN 전환 전 필수)**: 현재 역방향 수집 루프는
   한 페이지(30봉)가 시간창 가드로 전부 걸러져 `got==0`이면 즉시 break한다. J에선 정규장 봉만 와서
   무해하지만, UN은 NXT 장전(08:xx)·장후(15:4x~) 봉이 섞여 **정규장 봉이 아직 더 과거(위)에 남아있는데도**
   장 밖 봉만 담긴 페이지에서 조기 종료해 정규장 봉을 놓칠 수 있다. 전환 시 break 조건을
   "rows는 있는데 정규장 got==0이면 계속"으로 바꾸거나, 09:00 도달(`earliest<=SESSION_OPEN`) 기준으로만
   종료하도록 보강할 것. (현 상태=J 고정이라 잠복, 코드리뷰 2026-06-21 지적분.)

## 리스크 / 주의
- **시간창 가드 의존**: UN 전환의 안전성은 `SESSION_OPEN~CLOSE` 필터에 달려 있다. NXT 시각 체계가
  예상과 다르면(예: 초 단위·다른 필드) 가드가 새므로 STEP 1에서 반드시 확인.
- **거래량 누적 함정**: 분봉 거래대금은 `close×vol`(분당) 근사다(KIS 1분봉에 거래대금 필드 없음).
  UN에서도 동일 방식 — vol이 UN 누적분당이면 OK, 누적값이면 차분 필요(radar.aggregate_minute_bars 확인).
- **reignition은 표시 전용(score_raw=0)** 이라 통계 오염 위험은 낮지만, 텔레그램 재반등 알림 트리거이므로
  오탐 시 알림 스팸 가능 → STEP 1 검증을 건너뛰지 말 것.
- 분봉 미전환 상태로 두어도 **시스템은 정상 동작**(거래대금/유니버스는 이미 UN). 분봉 UN은 재반등
  10분봉 거래대금을 통합값으로 정밀화하는 마무리 단계일 뿐, 급하지 않다.

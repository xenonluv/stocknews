## 0. 먼저 알아둘 것 (중요)

- **저장소 주소**: `git@github.com:xenonluv/stocknews.git` (원격 이름 = `origin`, 브랜치 = `main`)
- **자동 커밋이 따로 돈다**: 장중에는 `publish.py`(cron)가 10분마다 `web/data/radar.json` 같은
  **데이터 파일을 자동으로 커밋·푸시**합니다. 그래서 **내가 푸시하기 직전에 원격이 먼저 바뀌어 있을 수 있음** → 아래 1번 "최신화"가 중요합니다.
- **비밀키는 절대 커밋되지 않음**: `.env`, `web/.env.local`, `.kis_token.json` 등은 `.gitignore`로 막혀 있습니다(신경 안 써도 됨).
- **작업 폴더**: 모든 명령은 `~/stocknews` 안에서 실행합니다.

```bash
cd ~/stocknews
```

> ⚠️ 터미널에 명령을 칠 때 **맨 앞에 `!`(느낌표)를 붙이지 마세요.** (히스토리 오작동 방지)

## 1. 원격 최신화 (푸시 전 항상)

cron이 먼저 올린 데이터 커밋과 충돌하지 않도록, **내 변경을 잠깐 치워두고(rebase) 원격을 먼저 받아옵니다.**

```bash
cd ~/stocknews
git pull --rebase --autostash
```

- `--autostash`: 아직 커밋 안 한 수정 사항을 자동으로 잠깐 빼뒀다가 다시 얹어줍니다.
- 충돌(conflict) 메시지가 나오면 → 아래 **8. 문제 해결** 참고.

---

## 2. 무엇이 바뀌었는지 확인

```bash
git status            # 바뀐 파일 목록 (빨강=아직 안 담음 / 초록=담음)
git diff              # 구체적으로 어디가 바뀌었는지 (q 키로 빠져나옴)
```

## 3. 올릴 파일 담기 (staging)

**코드/문서만** 담고, 임시 메모(`*.md` 낙서, 실험 파일)는 빼는 것을 권장합니다.

- 바뀐 코드 전부 담기(권장, 가장 간단):

```bash
git add -A
```

- 특정 파일만 골라 담고 싶을 때:

```bash
git add scripts/radar.py web/components/radar/SuspectCard.tsx CLAUDE.md
```

- 실수로 담은 파일 빼기(커밋은 유지, staging만 해제):

```bash
git restore --staged 파일이름
```

> 담은 결과는 `git status`로 다시 확인하세요(초록색이 올라갈 파일).

---

## 4. 커밋 (변경 확정 + 메시지)

메시지는 **무엇을 왜 바꿨는지** 한 줄로 적습니다(한글 OK).

```bash
git commit -m "레이더: 폭발/식음/반등 정의 개편 + 죽은 코드 정리"
```

여러 줄로 자세히 쓰고 싶으면 `-m`을 두 번:

```bash
git commit -m "레이더: 폭발 정의 22%+거래량90% 개편" -m "- 식음/반등 게이트 교체, Kimi 경로 제거, /forecast=당일 폭발 리스트"
```

## 5. 푸시 (GitHub로 올리기 → 사이트 자동 반영)

```bash
git push origin main
```

- 성공하면 끝. **약 30초 뒤 https://stocknews-cyan.vercel.app 가 새 코드로 다시 빌드**됩니다.
- `rejected`(거부) 메시지가 나오면 = 그새 cron이 또 올린 것 → **1번(pull --rebase)을 다시 하고 5번 재시도**:

```bash
git pull --rebase --autostash && git push origin main
```

## 8. 문제 해결 (자주 겪는 상황)

| 상황 | 해결 |
|------|------|
| `git push`가 `rejected ... fetch first` | 원격이 앞서 있음 → `git pull --rebase --autostash` 후 다시 `git push origin main` |
| `pull --rebase` 중 `CONFLICT` (충돌) | 보통 `web/data/radar.json` 등 데이터 파일 충돌. 내 코드가 맞으면 → `git checkout --theirs web/data/radar.json && git add web/data/radar.json && git rebase --continue` (데이터는 cron이 곧 다시 만듦). 잘 모르겠으면 멈추고 도움 요청. |
| rebase가 꼬여서 되돌리고 싶다 | `git rebase --abort` (rebase 시작 전 상태로 복귀) |
| 커밋 메시지를 잘못 썼다(아직 push 전) | `git commit --amend -m "새 메시지"` |
| 방금 한 커밋을 통째로 취소(아직 push 전, 수정 내용은 보존) | `git reset --soft HEAD~1` |
| 비밀키(.env 등)를 실수로 담았나 걱정 | `git status`에 `.env`/`.env.local`/`.kis_token.json`이 보이면 안 됨(보이면 멈추고 도움 요청) |

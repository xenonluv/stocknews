## 1. 재반등 레이더 조건 개편 (`573ff20`, `7d92d52`)

기존 "급등 후 식은 종목(fade)" 위주에서 **재매집·재반등(reaccum)** 감시로 확장.

- **모집단(reaccum)**: 13일 내 폭발(거래대금 1천억+ · 고가 +13%) → 식음 → MA20 위 → 투신(ivtr) 매집 → **재반등 10분봉**.
- **재반등 게이트 종가 기준 완화**: 고가 기준 → **종가 등락률 기준**(`REACCUM_CHANGE_MIN=-4.0`, `MAX=10.0`)으로 변경. (HL만도 같은 식은 종목이 누락되던 문제 해결)
- **재반등 봉 탐지**: `reignition_bars(bars, body_pct_min, value_min)` — 몸통 2%+ · 거래대금 30억+ 조건을 만족하는 **모든 10분봉**을 반환. 스캔은 그중 몸통 최대 봉을 대표로 쓰고 `reignition_bars` 필드로 전체를 실음.
- `score_raw=0` + `visible_experimental=True`로 격리 — 화면엔 보이되 **core 통계·가중치 튜닝엔 미반영**(드리프트 방지).

거래대금을 과소집계해 폭발 게이트(1천억+)에서 "매집 급등주"가 탈락하는 false negative 발생.

**조치 (A안 — 통합 UN 전환 + 실측 임계 재산정)**:
1. `kis_client.py` **가격=J / 거래대금·수급=UN 분리**: UN 종가는 NXT 시간외가 섞여 KRX 공식 종가와
   1~6% 어긋남(실측, 화신 6/19 J 14,330 vs UN 13,500) → 가격에 쓰면 MA·등락률·익일평가 왜곡. 그래서
   `daily_prices`/`price_now`는 **기본 J 유지**, `MONEY_MARKET=os.environ["KIS_MARKET"]||"UN"` + 병합
   헬퍼 `daily_prices_jmoney_un`/`price_now_jmoney_un`(J 가격 + UN 거래대금/거래량 덮어쓰기, 2콜)을 신설해
   레이더가 사용. `investor_trade_daily` 기본 UN(수급은 가격 아님). `minute_bars_today`는 정규장 시간창
   가드(`SESSION_OPEN~CLOSE`) 추가 후 **잠정 J 유지**(NXT 장전/야간 혼입 방지, 거래일 실측 검증 후 UN).
2. **순위 API(volume-rank)는 UN 거부**(J·NX만, 실측) → `value_rank_union()` 신설(J∪NX 합집합,
   거래대금 합산). `radar.py` 유니버스가 이를 사용 → NXT-헤비 종목 누락 차단.
3. UN/J 배율 실측(`nxt_calibration.py`, 1회성): 게이트 구간 중앙값 ~1.5배, **투신(ivtr)은 ~1.0배**
   (투신은 KRX 전용). → 폭발 하한 `EXPLOSION_VALUE` 1천억→**1,500억**, explosion 점수 스케일
   1천억~1조→1,500억~1.5조 ×1.5. **flow(투신) 점수는 불변**.
4. `radar.json` `params.market` 기록(레짐 마커, publish.py가 history에 전파).
5. 백테스트 재기준화 **불필요 확정**: `radar_backtest.py`는 절대 거래대금으로 bin하는 통계 없음

   `radar_weights.json` 미생성 상태라 아카이브 대상도 없음.

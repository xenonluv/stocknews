# 카드 뉴스 노출 + 3분봉 문구 제거 — 설계

- 날짜: 2026-05-31
- 목표: ① 시그널 카드(목록)에 관련 뉴스 제목 2건을 간략 노출, ② 사용자에게 보이는 "3분봉/골든크로스" 문구를 모두 제거(카드·홈 섹션 설명·상세).
- 비목표: 스크리너 C조건(3분봉 골든크로스) **판정 로직 자체는 그대로 유지**한다. 시그널/후보 등급 산정 기준은 변경하지 않고, 사용자 노출 문구만 정리한다.

## 배경

`SignalCard`는 현재 요약문(`summary`)과 "관련 뉴스 N건" 건수 힌트만 보여주고, 실제 뉴스 제목은 카드 클릭 후 상세페이지 `NewsList`에서만 노출된다. 요약문에는 "3분봉 갓 골든크로스(이격도 %)" 같은 3분봉 문구가 들어 있다. 사용자는 (1) 카드에서 바로 뉴스 제목을 보고 싶어 하고, (2) 3분봉 관련 문구가 노출되지 않기를 원한다.

## A. 카드에 뉴스 제목 2건 노출

- `toSignalCardProps`가 `news` 상위 2건(`{title, sentiment}`)을 카드 props로 전달.
- 카드 요약문 아래에 작은 뉴스 블록: 📰 아이콘 + 제목 2줄(각 1줄 말줄임 `line-clamp-1`). 호재/악재는 점(•) 색으로만 가볍게 구분(과한 뱃지 지양).
- **중첩 앵커 금지**: 홈에서 카드 전체가 상세로 가는 `<Link>`로 감싸져 있으므로, 카드 안 뉴스 제목은 **링크가 아닌 일반 텍스트**로 렌더한다. 개별 기사 링크 클릭은 상세페이지 `NewsList`(기존)에서 제공.
- 기존 "관련 뉴스 N건" 힌트는 이 뉴스 블록 헤더("📰 관련 뉴스 N")로 흡수.
- `news`가 없으면 블록 미표시(빈 상태 안전).

## B. 3분봉/골든크로스 문구 제거 (사용자 노출부 전부)

1. `scripts/publish.py` 요약문 생성:
   - 시그널: `[시그널] 일봉 {국면} · 재료 {호악재}(중요도 N). 일봉 국면을 함께 확인.` (← 3분봉 갓 골든크로스/이격도 절 삭제)
   - 후보: `[후보] 재료+거래대금 포착(중요도 N, 호악재) · 일봉 {국면}.` (← "3분봉 골든크로스 진입신호는 아직 대기 중" 삭제)
   - `gc_txt`/이격도 변수는 요약문에서 미사용 처리.
2. `web/app/page.tsx`:
   - 상단 부제: `스크리너(거래량·상승 이력 + 재료) + AI 분석 결과` (← "+ 3분봉 골든크로스" 삭제)
   - 📌 시그널 desc: `거래량·상승 이력 + 재료 포착` (← "3분봉 골든크로스" 삭제)
   - 👀 후보 desc: `재료 + 거래대금 포착` (← "3분봉 진입신호 대기 중" 삭제)
3. `web/data/signals.json`: 새 요약문으로 재생성.

> 코드 주석/내부 변수(C조건, `gc_recent`, `disparity_pct`)는 로직이므로 남긴다. 제거 대상은 **화면에 보이는 한국어 문구**뿐.

## 데이터 흐름 영향

`publish.py → signals.json → repository.ts → /api/signals → SignalCard/NewsList`. 타입 `SignalPost.news`는 이미 존재(직전 작업). 카드 props 매핑(`toSignalCardProps`)만 확장.

## 검증

1. `python3 scripts/publish.py --dry-run --names ...` → `/tmp/publish_preview.json` 요약문에 "3분봉" 미포함 확인.
2. 미리보기를 `web/data/signals.json`에 반영 후 `cd web && npm run build`.
3. 로컬 prod 서버(`PORT=숫자 npm start`)로:
   - 홈(`/`): 카드에 뉴스 제목 2건 노출, 부제/섹션 설명에 "3분봉" 미포함.
   - 상세(`/signals/{id}`): 기존 NewsList 정상.
   - HTML grep으로 "3분봉" 0건 확인.
4. 커밋 → `git push origin main` → Vercel 반영 후 라이브에서 동일 확인.

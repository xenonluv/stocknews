## 0. 이 시스템이 무엇인가 (1분 이해)

국내 주식 **매매 시그널 분석 사이트**입니다. 두 부분으로 나뉩니다:

```
🖥️ Mac Studio (집, 24시간 ON)             ☁️ Vercel (인터넷, 자동)
─────────────────────────────             ──────────────────────
파이썬 스크립트가 15분마다 자동 실행:        GitHub push를 감지해
 ① 네이버에서 거래대금/상승 상위 + 뉴스 수집   사이트를 자동 재배포
 ② 3분봉 골든크로스 등 3조건 분석             (Mac은 Vercel 설치 불필요)
 ③ 결과를 signals.json에 기록
 ④ GitHub에 push  ──────────────────────▶  https://stocknews-cyan.vercel.app
```

- **Mac이 하는 일**: 분석 + GitHub에 push (파이썬 + git).
- **사이트(Vercel)**: GitHub에 push되면 30초 내 자동 갱신. **Mac에 따로 설치할 것 없음.**
- **장중(평일 09:00~15:30)** 에만 의미 있음 — 3분봉은 장이 열려야 갱신됨. Mac은 24시간 켜두고, 스케줄러가 장중에만 작동.

핵심 원리: **분석은 표준 파이썬만 사용 → 추가 설치 거의 없음.** AI(Codex) 무거운 분석은 자동 루프에 없음(안정성). 자세한 구조는 같은 폴더의 `CLAUDE.md` 참고.

## 3. 시스템 내려받기 (clone)

### 3-1. GitHub 접속용 SSH 키 만들기 (push 자동화에 필요)
```bash
# 키 생성 (엔터 3번 — 비밀번호는 빈 칸으로 두어야 자동 push 가능)
ssh-keygen -t ed25519 -C "macstudio-stocknews" -f ~/.ssh/id_ed25519_stocknews -N ""

# 공개키 출력 → 복사

```
- 출력된 `ssh-ed25519 ...` 전체를 복사 → **GitHub.com → Settings → SSH and GPG keys → New SSH key** 에 붙여넣고 저장.

SSH 설정에 키 등록:
```bash
cat >> ~/.ssh/config <<'EOF'

Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_stocknews
  IdentitiesOnly yes
EOF

# 연결 테스트 (처음엔 yes 입력)
ssh -T git@github.com
```
- `Hi xenonluv! You've successfully authenticated...` 가 나오면 성공.

## 6. 24시간 자동 가동 설정

### 6-1. 시간대를 한국으로 (중요)
스케줄이 한국 장중에 맞으려면 Mac 시간대가 서울이어야 합니다.
```bash
sudo systemsetup -settimezone Asia/Seoul
date    # ... KST ... 확인
```
(또는 시스템 설정 → 일반 → 날짜 및 시간 → 시간대: 서울)

### 6-2. Mac이 잠들지 않게 (24시간 ON)
```bash
sudo pmset -a sleep 0           # 시스템 잠자기 끔
sudo pmset -a disablesleep 1    # (선택) 강제로 깨어있게
```
(또는 시스템 설정 → 디스플레이/에너지 → "자동으로 잠자기 안 함")

### 6-3. 자동 실행 등록 (cron — 가장 간단)
평일 09:00~15:45, **15분마다** 자동 분석+게시:
```bash
# python3 경로 확인 (보통 /usr/bin/python3)
which python3

# crontab 편집기 열기
crontab -e
```
편집기에서 **맨 아래에 아래 한 줄**을 추가(경로의 `사용자명`은 본인 것으로, python3 경로는 위에서 확인한 값으로):
```
*/15 9-15 * * 1-5 cd /Users/사용자명/stocknews && /usr/bin/python3 scripts/publish.py --max-candidates 8 --names 한온시스템 삼성전자 현대차 NAVER LG전자 삼성전기 노타 코칩 로보스타 로보티즈 >> /tmp/publish.log 2>&1
```
- 저장: (vi 편집기면) `Esc` → `:wq` → 엔터. (nano면 `Ctrl+O`→엔터→`Ctrl+X`)
- 확인: `crontab -l`

> 📌 **macOS 권한 한 번 허용 필요**: 시스템 설정 → 개인정보 보호 및 보안 → **전체 디스크 접근 권한**에서 `cron`(또는 터미널)을 추가/허용해야 cron이 정상 동작합니다. (안 그러면 조용히 실패)

### 6-4. (더 견고한 대안) launchd — 재부팅에도 자동 시작
cron이 불안하면 launchd 사용. 아래 파일을 만듭니다:
```bash
mkdir -p ~/Library/LaunchAgents
cat > ~/Library/LaunchAgents/com.stocknews.publish.plist <<'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key><string>com.stocknews.publish</string>
  <key>ProgramArguments</key>
  <array>
    <string>/bin/bash</string><string>-lc</string>
    <string>cd "$HOME/stocknews" &amp;&amp; /usr/bin/python3 scripts/publish.py --max-candidates 8 --names 한온시스템 삼성전자 현대차 NAVER LG전자 삼성전기 노타 코칩 로보스타 로보티즈 >> /tmp/publish.log 2>&1</string>
  </array>
  <key>StartInterval</key><integer>900</integer>
  <key>RunAtLoad</key><true/>
</dict>
</plist>
EOF

# 등록 (15분마다 24시간 실행 — 장외엔 변경이 없어 push 안 함, 안전)
launchctl load ~/Library/LaunchAgents/com.stocknews.publish.plist
```
- launchd는 15분마다 24시간 돌지만, 장외/변경 없을 땐 `publish.py`가 **push를 건너뛰므로** 사이트·Vercel에 영향 없습니다.
- 해제: `launchctl unload ~/Library/LaunchAgents/com.stocknews.publish.plist`

> cron **또는** launchd 중 **하나만** 쓰세요(둘 다 켜면 중복 실행).

## 9. 문제 해결 (Troubleshooting)

| 증상 | 원인 / 해결 |
|------|------------|
| `python3: command not found` | `xcode-select --install` 다시 실행. 경로는 `which python3`로 확인 후 cron에 반영. |
| `.env` 키 오류 / 뉴스 수집 0건 | `.env`의 `NAVER_CLIENT_ID/SECRET` 확인. `cat .env`로 값 점검. |
| `Permission denied (publickey)` (push 실패) | SSH 키 미등록. 3-1 단계 재확인. `ssh -T git@github.com` 테스트. |
| cron이 안 돎 | macOS **전체 디스크 접근 권한**에 cron/터미널 허용(6-3 참고). 또는 launchd(6-4) 사용. |
| 사이트가 안 바뀜 | `tail /tmp/publish.log`에서 `push` 성공 여부 확인. push됐는데 안 바뀌면 Vercel 대시보드에서 배포 상태 확인. |
| 결과가 늘 0건/그대로 | 주말·장외 시간엔 정상. 평일 장중(09:00~15:30)에 확인. |
| `git push` 가 비밀번호 물어봄 | SSH 대신 HTTPS로 clone된 것. `git remote set-url origin git@github.com:xenonluv/stocknews.git` 로 교체. |

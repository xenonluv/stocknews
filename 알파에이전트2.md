# 알파 에이전트 v2 (agent_alpha) — 전진수집기 + 실시간 재료·찌라시 추론 (전부 한 번에 빌드)

> 상태: **설계 문서(미구현)**. `알파에이전트.md`(v1, 재료·찌라시 추론 사이드카)를 확장 —
> **백테스트 불가능한 신호(14:30↑ 5분 스파크·거래원 키움집중·투자자별)를 매일 라이브로 적재해 전진검증하는
> "전진수집기"** 를 더해, v1의 LLM 추론과 **같은 (종목,날짜) 레코드**로 합친다.
> 코어와 완전 격리된 `agent_alpha/` 한 폴더. 효과 없으면 `rm -rf agent_alpha` → 코어 0 영향.

---

## 1. 왜 만드는가 (이번 세션 검증이 낳은 동기)

가설 **"고(高) 2일누적 유통회전율 + 음봉 → 익일 상승"** 을 824종목 × 200일 = **157,527 샘플**로 백테스트한 결과:

- **반증**: 400%+ 2일누적 회전율 음봉 → 익일 상승 **30%**, 평균 **−3.4%**(다스코류 극단 구간). 회전율 높을수록 더 나빴다.
- **그러나 핵심 신호를 못 넣었다**: 오너가 실제로 보는 **14:30~장종료 5분봉 양봉 스파크(≥2, 몸통%≥2%)**,
  거래원 키움 집중도, 투자자별(외국인/기관) 순매수는 **과거 데이터가 존재하지 않아** 백테스트 자체가 불가능:
  - KIS 당일 1분봉 = **당일만**. 네이버 fchart = **최근 ~6세션만**(시·고·저 null, 종가만).
  - 거래원(`inquire_member`) = **당일 스냅샷만**(과거일 조회 불가).
  - 투자자별(`investor_daily`) = **최근 ~30일만**.

→ **이 신호들은 "오늘부터 매일 라이브로 찍어 두고 익일에 채점"하는 전진수집 외에는 검증할 길이 없다.**
여기에 v1의 "왜 움직이나·진위·조작위험" LLM 추론을 같은 레코드에 붙이면, 몇 주 뒤
**"음봉 + 14:30 스파크 + (키움집중 or 외국인순매수) + (실재료 vs 찌라시펌프) → 익일"** 을 *진짜로* 검증할 수 있다.

> ⚠️ **자동매매·수익 보장이 아니다.** 스파크 신호는 *미증명 가설*이고(과거 반증된 건 turnover-음봉 극단뿐),
> 공개신호 엣지는 얇으며 단일 약세 레짐·체결비용 한계가 있다. 가치는 ① 검증불가 신호의 전진검증 인프라
> ② 가짜(찌라시) 사살 ③ 라이브 맥락. 효과가 데이터로 안 나오면 폴더째 삭제가 정답.

## 2. 목적

1. **전진검증 데이터셋 구축** — movers의 정량 신호(회전율·스파크·받힘·수급·거래원·레짐)를 매일 1행으로 적재,
   익일 결과로 라벨 → calibrate가 "어떤 조합이 익일에 통하나"를 *입증된 통계*로 산출.
2. **실시간 맥락·진위·조작 추론(v1)** — 움직이는 종목의 catalyst·실재료여부·지속성·조작위험을 LLM이 판단,
   레드팀이 찌라시를 *용의자*로 역검증, 같은 레코드에 결합.
3. **정직한 자가검증** — 정량 가설밴드 + LLM confidence Brier를 익일 결과로 채점. n 부족 동안 전부 "관찰".
4. **삭제안전 실험** — 코어 무손상. 안 통하면 `rm -rf agent_alpha` + cron 줄 제거 = 끝.

## 3. 설계 원칙 — 격리 (위반 시 삭제안전성 붕괴)

1. **모든 신규 코드는 `agent_alpha/`(레포 루트) 안에만.** 코어(`scripts/`·`web/`) **수정 0**(예외: `.gitignore` 1줄).
2. **단방향 의존**: agent_alpha → 코어(읽기전용 `import` + 출력 파일 읽기)만. **코어는 agent_alpha를 import/참조 안 함.**
   → `grep -rn "agent_alpha" scripts web` **빈 결과**여야 한다(= 코어가 agent_alpha를 모름 = 삭제해도 안 깨짐).
3. **쓰기는 `agent_alpha/data/` 안에만.** `radar.json`·`radar_history`·`.explosion_registry.json` 등 코어 상태 **절대 미수정.**
   ⚠ `float_ratio.get_float_and_listed(code, cache=내부dict)`로 호출해 코어의 `data/float_ratio.json` 디스크쓰기도 회피.
4. **자체 cron**(`agent_alpha/install_cron.sh`, 네임스페이스 마커 `# AGENT_ALPHA_BEGIN/END`).
5. **출력 표면 분리**: 자체 텔레그램 "🧠 [알파]" + 자체 상태파일. 웹 수정 0.
6. **코어 랭킹 영향 0**: 알파는 *평행한 별도 의견*. 입증돼도 자동 반영 안 함(추후 사람이 수동 승격).

### KIS 접근 = Option A (확정)
agent_alpha가 `scripts/kis_client.py`·`scripts/float_ratio.py`를 **읽기전용 라이브러리로 import**한다.
- 삭제안전 불변: 원칙 #2의 grep 테스트는 *코어가 agent_alpha를 참조하는가*를 보는 것 → A는 agent_alpha→코어
  방향이라 grep은 계속 빈 결과. `rm -rf agent_alpha` 시 코어 무손상.
- 이유: KIS 토큰 발급은 **1분 1회 제한** — 자체 클라이언트를 또 만들면 `.kis_token.json` 캐시를 두고 충돌.
  코어 모듈을 import하면 토큰캐시·레이트리밋(`MIN_GAP=0.06`)·재시도(`EGW00201`)를 **공유**해 안전·최소코드.
- v1 doc의 "import 금지" 문구는 *목적(삭제안전)*만 보존하고 완화한다(README에 근거 명시). **단 import는 읽기 전용 사용만.**

## 7. 루프/패스 배선 (전부 한 번에 = 4잡 동시 구성)

- **장중 fast loop** `loop.py` (cron `*/10 9-15`): radar.json movers(트리거, 중복 reasoning 안 함) →
  `news`+`rumors` 수집 → `analyst` LLM 판단 → `redteam` 적대 반박(찌라시=용의자·근거 substring 추적) →
  `data/judgments/{date}.json` 기록 + 고신뢰·통과분만 `notify`(🧠 [알파]). 종목·일자 디둡.
- **EOD collect** `collect.py` (cron `40 15`, **마감 후 필수** — 분봉 당일만): bounded movers(`MAX_MOVERS`) 각각
  `quant.py`로 정량행 생성(캔들·2일회전율·`sparks_min`·`investor_daily`·`kis_extra`·`regime`) +
  당일 `judgments/{date}.json`의 LLM 판단 결합(재호출 X) → `forward/{date}.json`+`forward_samples.jsonl`(labeled:false).
- **익일 label** `label.py` (cron `10 9` 다음 거래일): `forward_samples.jsonl`의 미라벨·과거행에 익일 일봉으로
  `next_*`/`hit` 채움(`radar_backtest.next_day_bar` 미러, 정지봉 스킵). jsonl read-all→rewrite.
- **장후 calibrate** `calibrate.py` (cron `45 17`): 라벨 표본으로
  (a) **정량 가설밴드** = `(turnover_2d 밴드 × spark_1430 밴드 × close_strength 밴드 × 음봉여부)` → 익일 hit율·평균·n
  (= 백테스트가 못 만든 스파크 포함 통계), (b) **LLM 보정** = confidence별 Brier·prob_up별 적중률 → `calibration.json`.
  n 부족 동안 전부 "관찰". 입증된 밴드만 이후 텔레그램에서 강조.

## 8. 비용/레이트
- KIS 20콜/s(`MIN_GAP=0.06` 공유). mover당 ~16콜(분봉 페이징 지배). `MAX_MOVERS=30` → ~480콜≈24s, EOD 배치 충분.
- LLM·거래원·investor는 bounded mover set·(code,date) 캐시. float 7일 디스크캐시(읽기).
- 스파크는 **당일에만 신뢰 포착**(분봉 당일만). 백필 mover는 fchart 폴백(종가만→몸통% 불가 시 `spark_source="none"`·null, **날조 금지**).

## 9. 빌드 순서 (의존성 순 — 한 번에, 파일별)

**0. 스캐폴드**: `README.md`, `config.py`, `.gitignore`에 `agent_alpha/data/` 1줄.
**1. 데이터층**: `kis_extra.py` → `sources/{movers,regime,sparks_min,news,rumors}.py` → `sources/quant.py`.
**2. 수집/라벨**: `collect.py --dry-run`(정량행+labeled:false) → `label.py`(익일 채움).
**3. LLM 알파**: `agents/analyst.py` → `agents/redteam.py` → `loop.py --dry-run`(judgments) → `collect.py`에 judgment 결합.
**4. 알림/보정/자동화**: `notify.py` → `calibrate.py` → `install_cron.sh`(4잡 + `--uninstall`).
**5. 검증**(§10) → 커밋(코어 0 수정 확인).

## 10. 검증 (삭제안전 1순위)
1. `grep -rn "agent_alpha" scripts web` → **빈 결과**.
2. 워크트리 복제 `rm -rf agent_alpha` → `python3 scripts/publish.py --dry-run` + `cd web && npm run build` → **통과**.
3. `python3 -m py_compile agent_alpha/**/*.py` 전체.
4. `python3 agent_alpha/collect.py --dry-run` → `agent_alpha/data/`에만 생성, `radar.json`·registry·`float_ratio.json` mtime 불변.
5. `python3 agent_alpha/loop.py --dry-run` → judgments 기록, 텔레그램 미발송.
6. `agent_alpha/agents/redteam.py` 단위(가짜 찌라시 "작전 의심") · `calibrate.py` calibration.json 생성.
7. `agent_alpha/install_cron.sh --dry-run` 멱등 · `--uninstall` 네임스페이스 블록만 제거.

## 11. 정직한 한계
- 전진데이터는 **수 주~수개월** 쌓여야 결론(그 전 전부 "관찰"). 스파크 신호는 **미증명 가설**.
- 공개신호 엣지 얇음·단일 약세 레짐·체결비용·슬리피지 → **삭제안전 실험 구조가 정답**.
- 스파크는 같은날만 포착(데이터 제약), 거래원은 EOD 스냅샷(틱 이력 없음·창구≠주체).
- 코어 수익을 *자동으로* 올려주지 않음. 본질 = 검증불가 신호의 전진검증 + 가짜 사살 + 라이브 맥락.

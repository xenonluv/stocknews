# 알파 에이전트 (agent_alpha) — 실시간 재료·찌라시 추론 사이드카

> 상태: **설계 문서(미구현)**. 코어와 완전 격리된 별도 폴더 `agent_alpha/`로 만들어, 효과가 없으면
> 폴더째 삭제해도 기존 레이더에 0 영향이게 한다. 실험 성격.

---

## 1. 왜 만드는가 (배경·이유)

현재 시스템은 **"기록방식" backbone**이다 — `radar.py`가 룰로 탐지(폭발/youtong/재매집)하고,
`radar_backtest.py`가 매일 익일 결과로 채점·보정하며 통계를 누적한다. 정직하고 견고하지만 두 가지 공백이 있다:

1. **"왜" 움직이는지를 실시간으로 못 읽는다.** 종목이 튀어도 그게 *실적·공급계약 같은 실재료*인지,
   *단순 테마 편승*인지, *토론방·텔레그램 찌라시 펌프(작전)* 인지 룰만으론 구분이 약하다.
2. **AI 심층 분석이 수동(클릭)·1종목 단위**다. `/api/stock/{code}/ai`·`/ask`·`/phase`는 사용자가 눌러야
   돈다 — *지금 움직이는 종목에 자동·연속으로* 붙지 않는다.

즉 **기록의존(통계) + α(실시간 재료·찌라시 추론 지능)** 이 필요하다. 단, 코어가 이미 복잡하므로
새 지능을 코어에 박지 않고 **격리된 실험 모듈**로 분리해, 검증 후 가치가 없으면 흔적 없이 제거한다.

---

## 2. 목적 (무엇을 이루나)

- **실시간 맥락 보강**: 움직이는 종목의 "왜·진위·지속성·조작위험"을 자동으로 읽어 사람 판단을 보조.
- **가짜 사살(적대적)**: 찌라시를 *신호가 아니라 용의자*로 보고 레드팀 에이전트가 작전·허위를 역검증.
- **정직한 자가검증**: 에이전트의 라이브 판단을 매일 익일 결과로 채점·보정 → "입증된 신뢰도"만 신뢰.
  (루프는 엔진, **기록·보정이 그 엔진을 'vibes'에서 '검증된 알파'로 바꾸는 고삐**.)
- **삭제안전 실험**: 효과가 데이터로 안 나오면 `rm -rf agent_alpha` 로 즉시 폐기(코어 무손상).

> ⚠️ **자동매매·수익 보장이 아니다.** 공개신호 엣지는 얇고 단일 레짐·체결비용 한계가 있다. 가치는
> ① 가짜 사살 ② 라이브 맥락 ③ 정직한 검증이지, "에이전트 N개 = 수익"이 아니다.

---

## 3. 설계 원칙 — 격리(이게 핵심, 위반 시 삭제안전성 붕괴)

1. **모든 신규 코드는 `agent_alpha/`(레포 루트) 안에만.** 기존 `scripts/`·`web/` 코어는 **수정 0**.
2. **단방향 의존**: agent_alpha는 코어의 **출력만 읽음** — `web/data/radar.json`(현재 movers),
   `data/radar_history/*.json`, 배포된 공개 API(`/api/stock/{code}`·`/ask`·`/phase`), 네이버·텔레그램 공개 소스.
   **코어는 agent_alpha를 import/참조하지 않는다.** → `grep -rn "agent_alpha" scripts web` 가 빈 결과여야 한다.
3. **쓰기는 `agent_alpha/data/` 안에만.** `radar.json`·`radar_history`·`.explosion_registry.json` 등 코어 상태 절대 미수정.
4. **자체 cron**(`agent_alpha/install_cron.sh`, 네임스페이스). 삭제 = `rm -rf agent_alpha` + crontab 줄 제거.
5. **출력 표면 분리**: 기본 = 자체 텔레그램("🧠 [알파] …", 기존 🚨재반등/⚡youtong과 구분). 웹 수정 0.
   (선택: 나중에 삭제안전한 `web/app/alpha/` + `web/data/alpha.json` 페이지 — 웹 표시 원할 때만.)
6. **코어 랭킹에 영향 0**: 알파는 *평행한 별도 의견*. 입증돼도 자동 반영 안 함(추후 사람이 수동 승격 가능).

---

## 4. 완성 시 기능

1. **실시간 "왜 움직이나" 자동 해설** — mover 트리거 시 즉시 뉴스+찌라시 수집 → catalyst 한 줄.
2. **재료 진위·지속성 판정** — 실재료 vs 테마편승 vs 찌라시펌프 구분 + 익일 상승확률·신뢰도.
3. **찌라시 작전 경보(적대적)** — "떡상" 류 글을 레드팀이 *덫인가?* 로 역검증.
4. **재료 ↔ 수급·차트 정합성 교차검증** — 호재인데 매도세 등 앞뒤 불일치 플래그.
5. **실시간 텔레그램 알파 브리핑** — 고신뢰 판단만 `🧠 [알파] {종목}: 왜↑·지속성·조작위험·근거링크`.
6. **자가 채점·보정** — 매일 판단을 익일 결과로 채점 → confidence별 적중률·Brier 입증.
7. **(선택) 라이브 알파 웹 카드** — `/alpha`에 실시간 해설·진위·조작위험·근거.
8. **자유질문 확장** — "왜 올라?/이 찌라시 믿어도 돼?"에 수집 근거로 답(연속화된 /ask).
9. **다중 에이전트 병렬** — 동시 movers를 재료/수급/조작/지속성 다관점으로 동시 분석.

## 5. 구현 방법 (아키텍처)

### 5.1 폴더 구조 (전부 신규)
```
agent_alpha/
  README.md          # 정체·삭제법("rm -rf agent_alpha + cron 줄 제거 = 끝, 코어 무손상")
  config.py          # 임계·경로·LLM 모델·소스 목록 (자기완결)
  sources/
    movers.py        # web/data/radar.json 읽어 현재 youtong/폭발/재매집 movers (READ ONLY)
    news.py          # 종목 실시간 뉴스(네이버 공개 API)
    rumors.py        # 토론방 + 텔레그램 찌라시 수집(자체 최소 스크래퍼 — 코어 미의존)
  agents/
    analyst.py       # 1차 판단: {catalyst, real_likelihood, sustainability, manipulation_risk,
                     #            prob_up, confidence, evidence[]}  (LLM)
    redteam.py       # 적대적 검증: 찌라시 작전/허위·근거 추적성·look-ahead·이미 반영된 재료인가
  loop.py            # 빠른 루프(장중): movers→gather→analyst→redteam→record(+텔레그램). code·일자 디둡
  calibrate.py       # 느린 루프(장후): 기록을 익일 결과로 채점→confidence별 Brier·적중률 보정
  notify.py          # 자체 텔레그램(알파 전용 라벨·상태파일, 기존 telegram_notify와 별개)
  data/              # 자체 출력 전부 — judgments/{date}.json, calibration.json, snapshots/
  install_cron.sh    # 자체 cron(장중 빠른 루프 N분 + 장후 calibrate)
```

### 5.2 두 개의 루프
- **빠른 루프(장중, 이벤트/N분)** `loop.py`:
  `movers.py`로 radar.json의 현재 movers 추출(트리거 — 중복 탐지 안 함) → 각 신선 mover에 대해
  `news.py`+`rumors.py`로 실시간 수집 → `analyst.py` LLM 판단 → `redteam.py` 적대적 반박
  (찌라시=용의자, 인용은 원문 substring 추적) → **`data/judgments/{date}.json`에 기록** + 고신뢰·통과분만
  `notify.py`로 텔레그램. 종목·일자 디둡·캐시로 비용 게이트(레이더 트리거 종목에만 reasoning).
- **느린 루프(장후, 일)** `calibrate.py`:
  어제 judgments를 익일 일봉(네이버/공개 소스)으로 채점 → confidence-band별 Brier·적중률 → `calibration.json`.
  **입증(잘 맞는 band)된 판단만** 이후 텔레그램/표시에서 강조(그 전엔 '관찰' 라벨).
  = 코어 `ai_click_eval.py`/`phase_eval.py` 철학을 모듈 안에 자체 복제(코어 미의존).

### 5.3 데이터 소스 (전부 읽기 전용)
| 소스 | 용도 | 참고(코어의 유사 구현 — 로직 참조용, import 금지) |
|------|------|-----|
| `web/data/radar.json` | 현재 movers(트리거) | — |
| `data/radar_history/*.json` | 과거 신호·결과 | — |
| 네이버 종목뉴스/토론방 | 실시간 재료·찌라시 | `web/lib/stock/rumors.ts`, `team1_collect.py` |
| 텔레그램 공개채널 | 찌라시 | `scripts/telegram_news.py` |
| 공개 API `/ask`·`/phase`·`/ai` | (선택) 재사용 | `web/lib/stock/{ask,phase,ai}.ts` |

### 5.4 LLM·시크릿
- Moonshot(Kimi) 재사용(읽기전용 env `MOONSHOT_API_KEY`) 또는 Anthropic. **env만 읽음 — 코드 결합 없음.**
- 텔레그램은 기존과 또 다른 상태파일 `agent_alpha/data/.alpha_notified.json`(종목·일자 디둡).

### 5.5 다중 에이전트(100개) 매핑
동시 movers 병렬 `analyst`+`redteam`, 판단마다 다관점 레드팀 N개(재료/수급/조작/지속성), 보정·모니터링 —
**전부 `agent_alpha` 내부**. 브루트포스 가중치 튜닝이 아니라 **"가설 생성 < 가짜 사살"** 비율.

## 6. 단계별 로드맵
- **1차 (MVP)**: `config.py`·`sources/movers.py`·`sources/rumors.py`·`agents/analyst.py`·`agents/redteam.py`·
  `loop.py --dry-run`·`notify.py`. → 현재 movers에 실시간 판단·텔레그램(수동 실행).
- **2차 (책임)**: `calibrate.py` + `install_cron.sh`(장중 루프 + 장후 채점). → 자가검증 누적 시작.
- **3차 (확장, 선택)**: `web/app/alpha/` + `web/data/alpha.json` 카드, 다관점 레드팀 병렬, 자유질문.

---

## 7. 검증 (삭제안전성이 1순위)
1. **코어 무의존 증명**: `grep -rn "agent_alpha" scripts web` → **빈 결과**.
2. **삭제안전 테스트**: 워크트리 복제에서 `rm -rf agent_alpha` 후 `python3 scripts/publish.py --dry-run`
   + `cd web && npm run build` → 그대로 통과.
3. `python3 agent_alpha/loop.py --dry-run` → 현재 radar.json movers로 판단 생성·출력(텔레그램 미발송), judgments 기록.
4. `agent_alpha/agents/redteam.py` 단위: 가짜 찌라시 샘플에 '작전 의심' 플래그.
5. `agent_alpha/calibrate.py` → 누적 judgments 채점·calibration.json 생성.
6. py_compile 전체 + `install_cron.sh` idempotent.

## 8. 정직한 한계
- 알파 엣지는 **얇고 시들 수 있다**(공개신호·단일 약세 레짐·체결비용·슬리피지). → **삭제안전 실험 구조가 정답.**
- **루프만으론 'vibes'** — calibration이 있어야 알파. 찌라시 적대처리·비용 게이트 필수.
- 코어 수익을 *자동으로* 올려주지 않는다. 사람 판단 보조 + 정직한 가설 검증 인프라가 본질.
